Three ways to restructure VetPath

Same brand, same verified content, same engines — three different information architectures, drawn from current Figma Community reference patterns. Click any card for a full-screen interactive preview.

A · Command Center

The app-first shell

Persistent left sidebar, top utility bar, dashboard of widgets on entry. The site behaves like serious planning software from the first second — marketing takes the back seat.

Ref: DashStack · BankDash · ByeWind kits
B · Guided Journey

One question at a time

No navigation until there's a plan. The front door is the interview itself — full-screen, calm, TurboTax-style. Overwhelm is the enemy; chrome disappears.

Ref: SAAS Onboarding · stepper flows
C · Mission Editorial

The institution

Story-led marketing site with editorial sections, impact numbers, and heavy trust signals — the app lives one click behind. Built to be shared with families, VSOs, and partners.

Ref: WeHope · NGO/Red Cross layouts

How to read these: each preview restructures the homepage plus the key screen it prioritizes. Palette, type (Fraunces/Inter), photography, and honesty rules are held constant so the comparison is purely structural. The current production site is a hybrid of B's intake with C's landing.
